# V7 Fast Track to EXE and APK

## Goal

Get the product ready as fast as possible for:
- Windows PC install with EXE
- mobile operation with APK
- real production, publication and selling flow

## Scope lock

Ignore non-critical polish until the core shipping flow is done.

## Fast track order

### Phase 1 - Mergeable V7 core
- Finish Studio V7 core flow
- Finish Website V7 core selling flow
- Validate one real project through readiness, freeze, publish and website visibility

### Phase 2 - Desktop packaging
- Prepare Studio backend/service startup for Windows
- Prepare desktop shell or operator UI entry point
- Build EXE packaging flow
- Define install/start/update path

### Phase 3 - Mobile operator app
- Expose only the operator actions needed from phone
- Build APK shell for remote operation
- Connect APK to the same real backend operations already validated in V7

### Phase 4 - Production mode
- Create first real content flow end to end
- Freeze publication package
- Publish to Website
- Verify product surfaces and selling flow

## What is not priority now
- extra polish that does not help production
- experimental channels beyond website
- cosmetic work that does not improve readiness, freeze, publish, sell or install

## Immediate execution order

1. Merge Studio V7
2. Merge Website V7
3. Add Windows EXE packaging branch
4. Add mobile APK operator branch
5. Run first real content product through the flow

## Definition of done for the first usable release

- Studio can run on Windows from EXE
- operator can trigger key actions from mobile
- a project can go from production to website publication
- website can surface and sell ebook and audiobook products
