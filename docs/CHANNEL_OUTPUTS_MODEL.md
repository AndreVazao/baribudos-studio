# Channel Outputs Model

## Objetivo
Definir o que cada canal deve receber quando a distribuição externa arrancar.

## Website próprio
Recebe:
- teaser
- página pública
- caminho de compra
- destaque comercial

## Amazon KDP
Recebe:
- ficheiro editorial final
- capa final
- copy comercial
- metadados prontos

## YouTube / YouTube Kids
Recebe:
- trailer
- vídeo principal
- título
- descrição
- thumbnails quando existirem

## Audiobook / outras plataformas
Recebe:
- áudio final
- capa
- copy
- readiness por canal

## Regra de arquitetura
O Studio prepara os outputs por canal. O Website não substitui nem gere esses outputs externos.
