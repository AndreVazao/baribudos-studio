# Studio Runbook

## Ordem prática
1. abrir frontend do Studio
2. validar backend
3. escolher projeto
4. preencher teaser e estado público
5. publicar para Website
6. validar páginas públicas

## Frontend
Pasta: `client/mobile-app/`

Scripts:
- `npm install`
- `npm run dev`
- `npm run tauri:dev`
- `npm run tauri:build`
- `npm run android:build`

## Regra de arquitetura
- O Studio manda.
- O Website recebe.
- Amazon e YouTube entram depois como destinos controlados pelo Studio.
